# ルーレットアプリ 要件定義

## 機能要件

### FR-01: 候補入力

- テキスト入力欄から候補を追加できる
- 1候補あたり最大6文字まで入力可能
- 入力中の文字数カウンターを表示する（`0 / 6` 形式）
- 最大文字数到達時にカウンターを強調表示する
- Enter キーまたは「+」ボタンで候補を追加できる
- 重複する候補は追加できない
- 空文字は追加できない
- 候補は最大16件まで登録可能

### FR-02: 候補管理

- 追加済みの候補をタグ形式で一覧表示する
- 各タグの「✕」ボタンで個別に削除できる
- 候補が2件未満の場合、スタートボタンを無効化する

### FR-03: ルーレット実行

- ボード背景に日本地図風の装飾（島4つ・雲3つ）を表示する
- 4x4（16枚）のパネルグリッドを地図の上に重ねて表示する
- パネルは裏向き（透明）状態からスタートし、裏面では背景の地図が透けて見える
- 同時に表を向くパネルは最大3枚
- パネルが自動的にランダムに表裏を切り替える
- 表になったパネルにはランダムな候補名を表示する

### FR-04: ストップ・結果表示

- 「ストップ」ボタンで回転を停止する
- 停止時、全パネルが裏返った後に当選パネル1枚だけが表になる
- 当選パネルにハイライトアニメーションを付与する
- 結果をオーバーレイダイアログで表示する
- 紙吹雪アニメーションを再生する

### FR-05: リトライ・画面遷移

- 結果表示後、「もう一回」ボタンで再抽選できる
- 「もどる」ボタンで入力画面に戻れる
- 入力画面に戻った際、候補リストは保持される

## 非機能要件

### NFR-01: パフォーマンス

- アニメーション制御に `requestAnimationFrame` を使用する（`setInterval` / `setTimeout` ループ禁止）
- DOM操作は `DocumentFragment` によるバッチ挿入で行う
- 紙吹雪は Canvas API で描画する（DOM要素の大量生成を避ける）
- アニメーション対象要素に `will-change: transform` を指定する
- GPU合成可能なプロパティ（`transform`, `opacity`）のみをアニメーションに使用する
- `box-shadow` のアニメーションは最小限に抑える

### NFR-02: アーキテクチャ・コード品質

- アプリケーション状態は `state` オブジェクトに集約する
- DOM参照は `dom` オブジェクトに初回キャッシュする
- イベント処理はイベント委譲パターンを使用する（リスナーの再登録を避ける）
- 画面遷移・リトライ時にアニメーションフレームとCanvasリソースを確実に解放する
- 即時関数（IIFE）でスコープを閉じ、グローバル汚染を防ぐ
- `'use strict'` モードで実行する

### NFR-03: アクセシビリティ

- 画面切り替えに `aria-hidden` 属性を使用する
- インタラクティブ要素に `aria-label` を付与する
- 結果ダイアログに `role="dialog"` と `aria-modal="true"` を設定する
- 結果テキストに `aria-live="assertive"` を設定し、スクリーンリーダーに通知する
- 結果表示時に OK ボタンへ自動フォーカスする

### NFR-04: レスポンシブ対応

- モバイル（480px以下）に対応したレイアウト調整を行う
- `100dvh`（Dynamic Viewport Height）でモバイルブラウザのアドレスバーに対応する
- フォントサイズに `clamp()` を使用し、画面幅に応じて可変にする
- コンテナ幅に `min()` を使用し、最大幅と画面幅の小さい方を採用する

### NFR-05: UI・デザイン

- 背景色は白（`#ffffff`）を基調とする
- ルーレットボードの背景は空と海のグラデーション（`#87ceeb` → `#2e86c1`）
- ボード内に日本地図風の島（緑グラデーション）と雲（半透明白）を装飾として配置する
- パネル裏面は透明とし、背景の地図装飾が透けて見えるようにする
- CSS カスタムプロパティでカラー・サイズ・シャドウを一元管理する
- フォントはシステムフォントスタック（`-apple-system`, `BlinkMacSystemFont` 等）を優先する
- オーバーレイに `backdrop-filter: blur` を適用する

### NFR-06: メタ情報

- `<meta name="description">` を設定する
- `<meta name="theme-color">` を設定する
- `lang="ja"` を設定する
- ビューポート設定（`width=device-width, initial-scale=1.0`）を含める

### NFR-07: セキュリティ

- ユーザー入力の表示には `textContent` を使用する（`innerHTML` による XSS を防止）
- 外部リソースへの依存を持たない（単一HTMLファイルで完結）

### NFR-08: ブラウザ対応

- 単一の HTML ファイルで動作する（ビルド不要）
- 外部ライブラリ・フレームワーク依存なし
- モダンブラウザ（Chrome, Safari, Firefox, Edge 最新版）で動作する
